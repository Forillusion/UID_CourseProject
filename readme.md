#### **一、 智能导航用户界面 (核心概念)**
*   **UI 地图 (Map For UI)**
    *   下载尺寸为 803 × 1404 像素的地图图像文件 “MapForUI.jpg”。
    *   地图图像中的 **1 个像素代表现实世界中的 1.7 米**。
    *   以地图的**左下角**作为现实世界坐标系的坐标原点 (0,0)。
    *   地图中**只有道路是可通行的**。
*   **UI 中的智能车辆 (IVs For UI)**
    *   每辆智能车辆 (IV) 用一个 **8米 × 3米** 的矩形来表示。
    *   每辆 IV 的可视化尺寸可以根据特定的缩放比例进行调整。

#### **二、 教学目的说明 (Pedagogical Awareness)**
*   本项目纯粹出于**教学目的**，而非实际应用。
*   **硬性要求 (Hard Requirements)：**
    *   **仅允许使用 Matlab**。不接受使用其他编程语言实现！
    *   必须有一个启动 UI 的**主脚本 (Main script)**。
    *   不需要进行任何临时或特殊的配置。
    *   只需运行主脚本即可启动 UI。
    *   启动后，**所有相关操作都必须且只能通过 UI 来完成**（不能再依赖命令行输入）。
    *   **尽量减少内置函数的使用**。
    *   允许使用：基本的数学内置函数、基本的图像 I/O（如 `imread` / `imshow`）和 Matlab UI 控件。
    *   **不接受使用其他高级内置函数！**

#### **三、 基本 UI 要求 (Basic UI Requirements)**
1.  只需运行主脚本即可启动 UI，此后 Matlab 命令行界面不应再参与后续操作。
2.  能够加载地图进行可视化显示。
3.  点击地图上的任意一点时，显示其**现实世界的位置**（即坐标）。
4.  能够将一辆（或多辆）IV 加载到地图中。
    *   UI 应提供一种方式来触发加载新 IV 的操作。
    *   通过**点击地图上的一个点**（即加载点）来加载 IV。
    *   IV 必须加载到道路上（因为道路是唯一可通行部分），因此**必须有一种机制来检查加载点是否有效**。
    *   如果加载点无效，应给出**错误提示**。
    *   如果加载点有效，则可视化显示包含新添加的 IV 以及已存在 IV 的地图。
5.  **支持调整方向的 IV 加载**：除了上述基本要求外，UI 还应提供一种调整 IV 方向的方法。
6.  能够从地图中**移除**一辆 IV。
    *   UI 应提供一种移除不需要的 IV 的方法，同时保持其他 IV 不受影响。
    *   在更新 IV 后，重新可视化地图。
7.  能够报告所有已加载 IV 的现实世界位置。
8.  能够显示地图上点击的**两个点之间**的现实世界距离。
9.  能够显示由一系列**连续点击的点**所构成的轨迹的现实世界长度。
10. **能够旋转地图**。
    *   UI 应提供一种指定旋转角度的方法。
    *   可视化显示按指定角度旋转后的地图。

#### **四、 可选 UI 要求 (Optional UI Requirements)**
*   **任务分配**：每名学生负责一个可选要求，该部分**独立评分**，至少需要选取4个。
*   **OR 1：道路骨架提取**
    *   UI 提供一种手动（部分）提取道路骨架的方法。
    *   UI 能够根据现实世界的配置可视化提取出的道路骨架。
    *   UI 能够仅可视化地图中与提取出的道路骨架相连的（大致）道路部分。
*   **OR 2：缩放与局部可视化**
    *   UI 可以根据可调的缩放比例调整 IV 的可视化尺寸。
    *   UI 可以根据可调的范围，仅可视化 IV 的局部地图。
    *   局部地图是以 IV 为中心的圆形，其半径为指定的范围。
    *   可视化的局部地图必须**严格遵守**指定的范围。
*   **OR 3：IV 加载时自动对齐方向**
    *   除了之前的 IV 加载基本要求外，IV 的方向应**自动与道路对齐**。
    *   能够以某种旋转方式可视化地图和 IV，使得 IV 看起来始终是朝上的（类似车载导航的“车头向上”模式）。
*   **OR 4：虚拟街景生成**
    *   能够生成与任意道路点相关联的虚拟街景，就好像是在该点拍摄的一样。
*   **OR 5：路径规划**
    *   给定地图上的任意两点，能够分别找到离它们最近的**道路点**。
    *   能够找到这两个道路点之间的**最短道路路径**，并在地图上可视化该最短路径。
