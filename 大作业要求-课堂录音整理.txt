
# 📝 ISE 333 课程项目：智能导航 UI (Intelligent Navigation UI) 讲解纪要

## 一、 项目背景与核心参数
本项目旨在通过开发一个智能导航 UI，让同学们体验和实践用户界面设计的过程。项目**纯粹出于教学目的**，而非实际商业应用。
*   **地图参数**：提供的 `MapForUI.jpg` 尺寸为 **803 × 1404 像素**。
*   **比例尺**：地图中 **1 个像素 = 现实世界中的 1.7 米**。
*   **坐标系约定**：
    *   **现实世界坐标系**：以地图**左下角**为原点 (0,0)，横向为 X 轴，纵向为 Y 轴。
    *   ⚠️ **Matlab 图像坐标系警告**：Matlab 中图像矩阵的原点在**左上角**，且 Y 轴方向**向下**。在计算和显示现实坐标时，**必须手动进行坐标系转换**，切勿搞混。
*   **可导航区域**：地图中**只有道路区域**是可通行的（凭常人视觉常识判断即可，无需钻牛角尖）。
*   **智能车辆 (IV)**：用 **8米 × 3米** 的矩形表示（类似大巴车）。在 UI 显示时，为了可视性，**允许按一定比例放大显示**，不必死板地按实际像素大小显示。

## 二、 硬性要求 (Hard Requirements) ⚠️ 红线
1.  **唯一语言**：**仅允许使用 Matlab**，不接受其他编程语言。
2.  **启动方式**：必须提供一个**主脚本 (Main script)**。将该脚本所在文件夹放在任意位置，直接运行即可启动 UI，**无需任何额外的路径配置或参数修改**。
3.  **纯 UI 交互**：UI 启动后，**所有后续操作必须且只能通过 UI 界面完成**，严禁在 Matlab 命令行中输入指令来辅助操作。
4.  **纯手写代码 (手搓)**：
    *   **严禁使用高级内置函数**（如图像叠加的 `patch` 函数、现成的图像旋转函数等）。
    *   **仅允许使用**：基础数学运算函数、基础图像 I/O (`imread` / `imshow`) 以及 Matlab 基础 UI 控件。
    *   所有图像叠加、旋转等操作，必须回归图像本质（矩阵），通过**手动修改像素矩阵**来实现，最后再用 `imshow` 显示。
5.  **核心考核点**：教授强调 **“重点在于可观察的实现过程，而不是最终结果”**。代码逻辑和实现过程必须清晰可见。

## 三、 基本 UI 要求 (Basic UI Requirements)
*(注：此部分为小组共同任务，成绩共享，一荣俱荣，一损俱损)*
1.  **加载地图**：提供机制将地图加载并可视化显示。
2.  **坐标显示**：点击地图上任意一点，UI 需显示该点的**现实世界坐标**。
3.  **加载智能车 (IV)**：
    *   提供机制启动“加载车辆”模式。
    *   用户点击地图加载车辆时，**必须有机制检查点击点是否在道路上**。
    *   若无效（非道路），给出错误提示；若有效，则在地图上可视化显示新车辆及已存在的车辆。
4.  **调整车辆朝向**：在加载车辆时，提供机制允许用户手动调整车辆的朝向（长方形的长边方向）。
5.  **移除智能车**：提供机制选择并移除不需要的车辆，且不影响其他车辆，移除后重新可视化地图。
6.  **报告位置**：能够显示/报告地图上所有已加载车辆的现实世界位置。
7.  **测距功能**：
    *   **两点距离**：点击两个点，显示它们之间的现实世界直线距离。
    *   **轨迹长度**：支持连续点击多个点，计算并显示这些点连成的轨迹的现实世界总长度。
8.  **旋转地图**：提供机制（如输入框）指定旋转角度，并通过**手动矩阵操作**实现地图的旋转显示。

## 四、 可选 UI 要求 (Optional UI Requirements)
*(注：共 5 个任务，每组 5 人，**每人独立负责 1 个**。此部分独立评分，旨在防止“搭便车”。教授强调：团队协作能互相节省精力，若极端利己主义只顾自己，反而会因缺乏基础模块支持而更吃力。)*

*   **OR 1：道路骨架提取**
    *   提供机制让用户手动（部分）提取道路骨架。
    *   能按现实世界配置，可视化提取出的抽象道路骨架（拓扑连接正确且物理位置吻合）。
    *   能仅可视化地图上附着在该骨架上的大致道路区域（“加肉”显示）。
*   **OR 2：缩放与局部可视化**
    *   提供可调的缩放因子，放大智能车的可视化尺寸。
    *   能以智能车为中心，可视化指定半径（米）的**圆形局部地图**。
    *   ⚠️ **严格要求**：显示的局部地图范围必须与用户指定的半径**严格吻合**，不能是随机或错误的尺寸。
*   **OR 3：车辆加载时自动对齐方向**
    *   在基本加载要求之上，车辆加载到道路点后，其朝向应**自动与该点道路的走向对齐**。
    *   支持“车头始终朝上”的显示模式（即车辆朝向固定向上，地图围绕车辆进行反向旋转显示）。
*   **OR 4：虚拟街景生成**
    *   给定任意道路点及朝向，基于平面地图和三维映射原理，生成该点的虚拟街景图像。（注：相关计算机视觉知识教授会在后续练习课中补充讲解）。
*   **OR 5：路径规划**
    *   给定地图上任意两点，能分别找到离它们最近的**道路点**。
    *   能计算并可视化这两个道路点之间的**最短道路路径**。（注：需手写如 Dijkstra 等图搜索算法，不能直接调包）。
